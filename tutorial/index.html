What is LIME Bootstrap (LBS)?

LBS is:

  • A way of building nice looking HTML things in LIME Pro without worrying too much
  • A framework of smart libraries
  • A open source project, free to contribute to
  • A platform and store for delivering apps
  • A version control system

A Model-View-View Model framework




Powered by knockout.js

A single page app like structure

Heart and soul of LBS:

  • KnockoutJS: The powerful engine behind LBS. Makes smart web GUIs a walk in the park.
  • Twitter Bootstrap: A fantastic web-framework which LBS adapts and extends. Go here for finding out everything about the elements you would like to use, such as inputs or buttons.
  • Font Awesome: Hundreds of icons ready to use. Available in any size or color as it's really a font!

    • Included smart tools:

  • Moment.js: Makes working with time and date a joy!
  • Underscore.js:Powerful and smart library which takes away the pain when working with data in JavaScript
  • jQuery.js:The defacto standard for working with the DOM. Seldom used in LBS, but there are occasions where jQuery is needed

    What's happening?

    Working with views

    • A view is a partial web-page, containing only part of the page
    • A view should never load any resources, such as JavaScript or CSS!
    • Contained in an .html-file, i.e "company.html"
    • The query-string "?ap=[viewname]" decides which view is loaded
    • LIME automatically sets "?ap=[Name of class of ActiveInspector]

    Bidings:

    • Bindings connect data and functions from the ViewModel to the View.
    • KnockoutJS supplies many bindings, found here
    • LBS supplies additional bindings, found here

    Your ActionPad will probably look like this:

    The red this is the debug log. She is our friend. She shows up if we make a bad mistake and tries to help us fix it. We can even have here around all the time to help us. Just open "_config.js" and set "debug=true". Do this now

    The console log says that "company is undefined". By default only the ActiveInspectors' data is loaded. Luckily we can ask for more data in "_config.js". Add a "person" node and a dataSource. There is a nice VBA function "ActionPadTools.GetCompanyContactData" which returns a related company record

            
            lbs.externalConfig = {
            /**
            Enable or disable the debug console 
            for the whole application
            **/
            debug: true,

            /**
            Configure special use cases,
            mainly when requiring additional datasources
            **/
            config:{
            'index' : {
            dataSources: [
            { type: 'localization', source: '' },
            ],
            autorefresh : false
        },

        'person' : {
        dataSources: [
        {type: 'activeInspector', source: ''},
        {type: 'localization', source: '' },
        {type: 'relatedRecord', source: 'company', view: 'name;postalcity;phone'},
        ],
        autorefresh : false
    }
}
}
